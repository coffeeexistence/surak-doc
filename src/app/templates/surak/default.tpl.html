Package Management

To install a package in Surak, run: bower install package-name --save
Then locate the package's folder in vendor/project-name.
Choose the file that you would like to include, which will usually be vendor/project-name/project-name.min.js.
Then add that to the "dependencies" array in surak.json, which is in the project root.

"dependencies" is where grunt looks when concatenating it's assets.
The order in which dependencies are loaded matter, so make sure to order them correctly!

File Structure
Out of all the files provided by Surak, you will only really need to focus on the ones in green.
build
dist
node_modules
src
vendor
bower.json
Gruntfile.js
package.json
surak.json

Dist:

Dist contains all production-ready files, including the html index, minified javascript and css, and images.
index.html is the only text file here that you will be modifying, everything else will be generated by grunt.

Src:

Src contains all of the javascipt and css source files.

Vendor:

Vendor contains all of libraries that your app depends on
This folder is the Bower installation directory, everything here should first be installed through
bower install package-name --save or by manually adding it to the dependencies array in bower.json.
Unless you are having issues with version conflicts, you don't need to have this in your app's repo.

Surak.json:

surak.json is currently used for managing which 3rd party libraries are concatenated into app.min.js.
